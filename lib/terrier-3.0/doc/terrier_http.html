Using the Web-based Terrier application

Using the Web-based interface

Terrier provides a simple Web-based interface. Firstly, the Web-based interface has slightly higher requirements than Terrier -- in particular, as JSPs are used, the full Java Development Kit (JDK) is required, instead of the JRE. To download the JDK, see Java downloads. Finally, ensure that the JAVA_HOME environment variable points to your JDK installation.

You can start the Web-based interface using the following command:

bin/http_terrier.sh

As usual, you can override the default etc/ folder using the TERRIER_ETC environment variable:

TERRIER_ETC=etc/wt2g bin/http_terrier.sh

Once started, the server can be accessed at http://localhost:8080/.

Customising look & feel

You can change the look and feel of the search results by editing src/webapps/simple/results.jsp, or the stylesheet in the same location. Changes made to results.jsp should have an immediate effect on the results.

If you wish to make another webapps folder, or started on a port other than 8080, you can override both on the command line.

TERRIER_ETC=etc/wt2g bin/http_terrier.sh 8000 /path/to/webapps/

Extending the Query Language

Terrier provides several PostFilters that are useful for interactive operation. For example, the SiteFilter is an implementation of the site: control from standard Web search engines, and ensures that returned results have a hostname matching the desired expression. The hostname is obtained from the url metadata from the MetaIndex. For example, adding site:com to the query will ensure that all results have URLs from the .com domain. For TREC or similar collections, the Scope filter permits a starting prefix on the docno document metadata from the MetaIndex.

Highlighting and Query-biased Summaries

SimpleDecorate is responsible for adding document metadata from the MetaIndex to the ResultSet. Decorate is a more involved decorator, which facilitates query-biassed summarisation, and highlighting of query term matches in the metadata.

Further Details

bin/http_terrier.sh invokes SimpleJettyHTTPServer, which starts a Jetty server on the port specified on the command line. The second command line argument is the path to a webapps folder. share/images is also mounted as /images directory.
